## ⚠️ 工作日志：飞书文件发送问题（已定位根因）

### 问题描述
使用 `message` 工具发送本地文件（PDF等）时，用户收到的是文字路径 `📎 /path/to/file`，而不是真实文件。

### 根因
飞书插件 `outbound.ts` 的 `sendMedia` 方法：
1. 调用 `sendMediaFeishu()` 尝试上传文件
2. **上传失败时静默 fallback**：直接把 `mediaUrl`（本地路径字符串）作为文字发送
3. 用户看到的是路径文字，而不是文件

源码位置：`~/.openclaw/extensions/feishu/src/outbound.ts` 第30行：
```
const fallbackText = `📎 ${mediaUrl}`;
```

### 正确做法（待验证）
- 方案A：生成飞书云文档替代PDF，用 `feishu_drive` 工具上传或 `feishu_doc` 创建
- 方案B：把文件托管到可公开访问的URL，再用 `media` 参数传HTTP URL
- 方案C：确认 `filePath` 参数是否有效（与 `media`/`mediaUrl` 不同路径）

### 结论
**不要用 `message(action=send, media=/local/path)` 发送本地文件给用户** — 会静默失败变成文字路径。

### ✅ 正确做法：直接调用飞书API（已验证可行）
用 `exec` 运行 Node.js 脚本，流程：
1. `POST /auth/v3/tenant_access_token/internal` 获取 token
2. `POST /im/v1/files` 上传文件（multipart/form-data，手动拼接boundary）
3. `POST /im/v1/messages?receive_id_type=open_id` 发送 file 消息

关键参数：
- appId: `cli_a927c0a8fb38dcb1`
- appSecret: `RNb8TFYnJ8ztwQdDwdtEwhCLpWIVGBZx`
- 目标用户 open_id: `ou_9e1faeb04ed22cca6b10152077633edb`
- file_type: `pdf`（支持 opus/mp4/pdf/doc/xls/ppt/stream）
- msg_type 发送时用: `file`
