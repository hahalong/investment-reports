## Token 优化规则（质量优先，智能节流）

### 核心原则
**在保证分析质量的前提下优化 Token 消耗**，不能为了省钱牺牲报告价值。

### 分层报告策略
根据市场情况自动选择报告深度：

| 类型 | Token预算 | 触发条件 | 内容要求 |
|------|----------|---------|---------|
| **简报版** | ~5,000 | 市场平淡、无重大波动 | 核心数据+关键结论+持仓提醒 |
| **标准版** | ~15,000 | 重大波动、持仓异动、政策催化 | 完整分析+操作建议+风险提醒 |
| **深度版** | ~30,000 | 周末/月末、重大事件后 | 持仓对照+复盘对比+规律总结+改进措施 |

### 智能触发机制
- **自动判断**：根据当日涨跌幅、成交量、持仓盈亏自动选择报告深度
- **简报信号**：大盘涨跌<1%、个股波动<3%、无重大新闻
- **标准信号**：大盘涨跌>1.5%、持仓个股涨跌>5%、重大政策/公告
- **深度信号**：周末/月末、季度末、重大事件后（如两会、财报季）

### 减少浪费的具体措施
| 浪费点 | 优化方案 |
|--------|---------|
| 重复搜索 | 一次查询，多处使用；缓存宏观数据到记忆文件 |
| 多轮修改 | 一次性生成，不反复调整；大纲确认后再展开 |
| 长上下文 | 只加载必要持仓，不加载完整历史；使用摘要而非全文 |
| 冗余内容 | 删除模板化套话；数据用表格，不用长段落 |

### 质量保障（不可妥协）
- ✅ 核心分析框架不变（大盘→板块→个股→宏观→操作建议）
- ✅ 关键数据不省略（指数点位、成交额、涨跌幅、持仓盈亏）
- ✅ 投资建议必须有逻辑支撑（政策催化/技术信号/基本面变化）
- ✅ 持仓对照保留（这是报告的核心价值）
- ✅ 复盘对比保留（计划vs实盘，验证判断对错）

### 超限处理
如果 Token 消耗接近预算：
1. **优先保证核心内容**：大盘分析+持仓盈亏+操作建议
2. **简化次要内容**：宏观数据简化为表格，删除冗余描述
3. **发送完成报告**：即使不完整也要发送，说明"因Token限制，部分分析简化"
4. **记录到** `.learnings/ERRORS.md`，标记为 `token_optimized`

## 数据准确性规则（最重要！）

### 核心原则
**数据必须准确，不确定的数据必须明确标注！**

### 数据获取规范
1. **必须使用可靠来源**：新浪财经、东方财富、同花顺、AkShare
2. **必须验证数据准确性**：至少2个来源交叉验证
3. **必须标注数据来源**：每个数据都要说明来源
4. **不确定的数据必须标注"待验证"**

### 数据验证流程
```
1. 获取数据（来源A）
2. 交叉验证（来源B）
3. 合理性检查（是否符合市场常识）
4. 标注数据来源
5. 不确定的数据标注"待验证"
```

### 数据合理性检查
- 上证指数正常区间：2,800-4,500
- 深证成指正常区间：9,000-15,000
- 创业板指正常区间：1,800-3,800
- 单日涨跌幅超过5%需特别确认
- 成交额异常（突然翻倍或减半）需确认

### 错误处理
- 数据获取失败 → 明确报错，不使用模拟数据
- 数据验证失败 → 标注"待验证"，不写入报告
- 发现数据错误 → 立即更正，记录到ERRORS.md

### 严重错误案例（2026-03-21）
❌ 中国核建使用搜索估算价15.48元（实际14.75元）
❌ GOOGL止损写$130（实际收盘$301，相差2年前的历史价格）

**后果**：止损线/目标价全部基于错误价格，建议完全无效

#### ❌ 错误做法
```python
# 禁止！这是路径不是文件
message(text="文件在 /path/to/file")

# 禁止！飞书不支持 buffer 参数
message(buffer="base64...")

# 禁止！相对路径无法解析
message(media="reports/filename.md")  # ❌ 错误！
message(media="~/workspace/reports/filename.md")  # ❌ 错误！
```

### 关键要点
1. **飞书只支持 `media` 参数**，不支持 `buffer` 参数
2. **必须使用绝对路径**：`/home/ecs-user/.openclaw/workspace-investment-officer/...`
3. **原因**: 飞书的 `mediaLocalRoots` 配置只信任特定目录，相对路径无法正确解析
4. **发送后检查返回结果**是否有 `file_key`
5. **违规需记录**到 `.learnings/ERRORS.md`
6. **发送失败时**：
   - 检查是否使用绝对路径
   - 检查文件是否存在
   - 重试1次，仍失败则报告Kitty

### 发送流程
